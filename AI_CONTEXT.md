## 🏗️ Philosophy and Execution Architecture
MicroCoreOS is a modular, asynchronous, and resilient system based on Clean Architecture principles.

- **Resilient Core**: The Kernel and Container are the stable heart. Plugin failures don't stop the system.
- **Native Registry**: The system inventory is now part of the Core, ensuring total observability.
- **Threading Model**: Plugins start in independent threads. The Kernel uses `RLock` for thread safety.
- **Concurrency Control**: The `event_bus` uses a `ThreadPoolExecutor` for efficient event handling.
- **Dependency Injection**: Plugins receive tools in the constructor. **Always check the 'Tools' section for method signatures.**

## 🚀 Execution and Development

- **Pydantic**: Pass your models to `http_server.add_endpoint` to auto-generate Swagger.
