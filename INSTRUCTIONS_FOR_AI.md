# 🤖 AI Agent Implementation Guide

## ⚠️ CRITICAL ARCHITECTURE RULES (DO NOT IGNORE)
1. **NO MANUAL INJECTION IN `main.py`:** The `Kernel` handles 100% of dependency injection via auto-discovery of the `tools/` and `domains/` folders. **NEVER** modify `main.py` to manually instantiate tools or plugins.
2. **NO HARDCODED IMPORTS:** Plugins must only request tools via their `__init__` arguments matching the tool's `name` property. Do not `import` tools directly into plugins.
3. **DO NOT APPLY FASTAPI/FLASK PATTERNS:** This is an Atomic Microkernel. There are no "Routers", "Controllers", or "Services". There are only Tools (Infrastructure capabilities) and Plugins (Domain business logic).

This guide is an "Interface Router" for AI agents. Do not read the whole document—jump to the section that matches your task.

## 🧩 New Domain Hierarchy
Ensure compliance with the folder structure:
- `domains/{name}/__init__.py`
- `domains/{name}/models/`: Pydantic schemas.
- `domains/{name}/migrations/`: `.sql` files for database migrations.
- `domains/{name}/plugins/`: Business logic.
*Domains MUST NOT import from each other.*

---

## 🤖 The AI-Driven Development Cycle
When building new data-driven features, always follow this Inside-Out process:

1. **Step 1: Domain Modeling (Pydantic)**
   - **Command:** `Create the Pydantic model ProductEntity in domains/products/models/product.py defining strict data types.`
   - **Why:** Establishes the source of truth for the domain's data structure.
2. **Step 2: Database Migration (Raw SQL)**
   - **Command:** `Using the ProductEntity model, create the SQL table creation script at domains/products/migrations/001_create_products.sql.`
   - **Why:** We prefer Raw SQL migrations generated by AI over ORMs. The `db` Tool will automatically execute pending `.sql` files sequentially on boot. PostgreSQL uses `$1, $2, $3...` placeholders.
3. **Step 3: The Use Case (Plugin)**
   - **Command:** `Create the CreateProductPlugin that receives db, reads the Request, and executes an INSERT using the model format.`
   - **Why:** The Plugin now has the table infrastructure ready to be used.

## ⚡ New Plugin (Single Feature)
**Goal**: Add logic using existing tools.
1. **Location**: `domains/{domain}/plugins/`
2. **Rule**: 1 File = 1 Feature.
3. **Usage Pattern**: Tools are injected via `__init__`. Register the handler directly in `on_boot`, implement logic in `execute(data, context=None)`.

```python
from typing import TYPE_CHECKING
from core.base_plugin import BasePlugin
from domains.products.models.product import ProductEntity

if TYPE_CHECKING:
    from tools.http_server.http_server_tool import HttpServerTool
    from tools.postgresql.postgresql_tool import PostgresqlTool
    from tools.logger.logger_tool import LoggerTool
    from tools.event_bus.event_bus_tool import EventBusTool

class CreateProductPlugin(BasePlugin):
    def __init__(self, logger: 'LoggerTool', event_bus: 'EventBusTool', http: 'HttpServerTool', db: 'PostgresqlTool'):
        # 1. Dependency Injection — save as instance attributes, no logic here
        self.logger = logger
        self.bus = event_bus
        self.http = http
        self.db = db

    async def on_boot(self):
        # 2. Registration Phase — register endpoints and subscriptions
        self.http.add_endpoint(
            path="/products",
            method="POST",
            handler=self.execute,
            tags=["Products"],
            request_model=ProductEntity
        )
        await self.bus.subscribe("order.created", self.on_order_created)

    async def execute(self, data: dict, context=None):
        # 3. HTTP handler — context available for set_cookie(), set_header(), set_status()
        try:
            product = ProductEntity(**data)
            product_id = await self.db.execute(
                "INSERT INTO products (name, price) VALUES ($1, $2) RETURNING id",
                [product.name, product.price]
            )
            await self.bus.publish("product.created", {"id": product_id})
            return {"success": True, "data": {"id": product_id}}
        except Exception as e:
            self.logger.error(f"Failed to create product: {e}")
            return {"success": False, "error": str(e)}

    async def on_order_created(self, data: dict) -> None:
        # Event subscriber — receives only data dict, no context.
        # For request() RPC, return a dict value instead of None.
        self.logger.info(f"Order received: {data}")
```

> [!TIP]
> **Hybrid Power**: Use `async def` for I/O tasks (database, network).
> If you have a heavy CPU task (video, crypto), use `def`. The Kernel will automatically offload `def` methods to a thread pool to avoid blocking the system.

## 🔧 New Tool (Infrastructure)
**Goal**: Add new technical capabilities to the system.
1. **Location**: `tools/{name}/`
2. **Structure**: Entry point MUST be `{name}_tool.py` at the root of the folder.
3. **Template**:

```python
from core.base_tool import BaseTool

class MyServiceTool(BaseTool):
    @property
    def name(self) -> str:
        return "my_service"

    def setup(self):
        # 1. Allocation Phase: Connect to external systems, load env vars.
        print("[MyService] Initializing...")

    def get_interface_description(self) -> str:
        # 2. Manual for the AI: Describe PURPOSE and CAPABILITIES with exact signatures.
        return """
        My Service Tool (my_service):
        - PURPOSE: Briefly describe what it does.
        - CAPABILITIES:
            - do_something(arg1, arg2): Description of what this method does.
        """

    def on_boot_complete(self, container):
        # 3. Optional: Executed when all tools are ready. Access other tools via container.get('name').
        pass

    def shutdown(self):
        # 4. Cleanup Phase: Close connections, stop threads.
        pass
```

## 🔄 System Lifecycle

### 🛠️ Tool Lifecycle
1. **`setup()`**: **Resource Allocation**. Connect to databases, read env vars.
2. **`on_boot_complete(container)`**: **Orchestration**. Executed when *all* tools are ready.
3. **`shutdown()`**: **Cleanup**. Close sockets, stop threads, save remaining state.

### 🧩 Plugin Lifecycle
1. **`__init__`**: **DI Phase**. Just save the requested tools. Don't perform logic here.
2. **`on_boot()`**: **Subscription Phase**. Register HTTP endpoints or EventBus subscribers.
3. **handler method** (e.g. `execute`): **Action Phase**. Called by HTTP or EventBus. Signature: `async def execute(self, data: dict, context=None)`.
4. **`shutdown()`**: **Cleanup**. Optional.

> [!IMPORTANT]
> **The Async Rule**: If you use `async def`, NEVER use `time.sleep()`. Use `await asyncio.sleep()`.
> If you must use a blocking library, use `def` and the Kernel will handle it.
